# FinancialReport Manual Drop-zone — P29B Scaffold

**Status:** OPERATOR_PENDING — awaiting real source files
**PIT Gate:** `filingDate` (NOT `periodEndDate`)
**entersAlphaScore:** false — this zone is intake-only, no scoring until P29-F dry-run gate passes

---

## Purpose

This directory is the manual drop-zone for Taiwan-listed company financial report data
collected by the operator from authorised public sources (e.g., TWSE MOPS, IFRS filings).

Files placed here are **NOT automatically imported**. They must pass the P29-F Dry-run Gate
before any corpus or scoring pipeline receives them.

---

## PIT Safety Rule

**Only `filingDate` may be used as the PIT gate.**

`periodEndDate` (e.g., Q1 ends 2025-03-31) is NOT the filing date and must never be used
as the availability anchor. A report for Q1-2025 filed on 2025-05-15 is only PIT-safe
from `filingDate = 2025-05-15` onward.

---

## What to Place Here

- One CSV or JSON file per source batch
- Each file must conform to `EXPECTED_SCHEMA.json`
- Each file must be listed in a `SOURCE_MANIFEST.json` (use `SOURCE_MANIFEST_TEMPLATE.json` as starting point)
- All files must be real, verifiable source data — no synthetic/mock rows

## What NOT to Place Here

- Any file with `DO_NOT_IMPORT` in the filename (those are templates)
- The `SOURCE_MANIFEST_TEMPLATE.json` file itself
- Files containing `approvalToken` (token must be generated by the import gate, not pre-filled)
- Synthetic, estimated, or forward-projected financial figures
- Files derived from broker forecasts or consensus estimates

---

## Import Flow (do NOT skip steps)

```
1. Operator places file(s) here
2. Operator fills SOURCE_MANIFEST.json (copy from template, remove _TEMPLATE suffix)
3. Operator runs operator QA checklist (docs/manual-data/financial-report/P29D_OPERATOR_FILE_QA_CHECKLIST.md)
4. P29-F Dry-run Gate validates schema, PIT gate, deduplication
5. If gate passes → import proceeds under separate approval
6. entersAlphaScore remains false until explicit P29-G gate
```
